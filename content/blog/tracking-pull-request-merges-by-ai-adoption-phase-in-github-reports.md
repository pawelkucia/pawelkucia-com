## Enhanced Insights with Adoption Phase Cohorts
GitHub has updated its Copilot usage metrics API to incorporate AI adoption phase cohorts. This latest enhancement now includes the total number of pull requests merged for each adoption phase within organization and enterprise reports.

## Why Track Merges by Adoption Phase?
Merging pull requests is a key indicator of software development progress and collaboration. By tying these metrics to AI adoption phases, teams can gain clearer visibility into how different levels of AI tool engagement impact their development workflows.

## Benefits for Organizations and Enterprises
With detailed reporting on merged pull requests by adoption phase, decision-makers can better understand AI-assisted coding's role in code integration and delivery. This fosters data-driven strategies for scaling AI tools and optimizing team productivity.

## Looking Ahead
As AI-powered tools like GitHub Copilot become more prevalent, comprehensive metrics help organizations measure their ROI and guide the evolution of developer workflows. Tracking merges by adoption phase is a significant step toward deeper operational intelligence.
