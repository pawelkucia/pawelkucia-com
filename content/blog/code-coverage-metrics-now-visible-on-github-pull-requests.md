## Code Coverage on Pull Requests

GitHub has rolled out a public preview feature that makes code coverage metrics visible directly on pull requests for all Code Quality users on github.com. This allows developers and reviewers to see an aggregate percentage of code coverage without leaving the PR interface.

## Why It Matters

Code coverage insights help highlight how well the code changes are tested. Having immediate visibility into coverage during the review process encourages better testing practices and supports stronger code health overall.

## What to Expect

This integration provides a streamlined way to evaluate the testing impact of new contributions, promoting higher quality and more reliable software development within teams.
