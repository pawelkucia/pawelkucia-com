## Introduction

GitHub Enterprise Cloud has reached a new milestone with the general availability of IP allow list enforcement across Enterprise Managed User (EMU) namespaces. This feature offers enterprises enhanced control over how user access is managed based on IP restrictions.

## What Are Enterprise Managed Users (EMUs)?

Enterprise Managed Users are identities managed by the enterprise within GitHub, providing a centralized way to handle user access and permissions. EMUs facilitate better governance and streamlined user administration across an organization.

## Enhancing Security with IP Allow Lists

With this update, enterprises can now apply GitHub’s native IP allow lists directly to EMU namespaces, tightening security by restricting access to approved IP addresses only. This reduces risk by ensuring that only users connecting from trusted networks can access sensitive repositories and resources.

## Implications for Enterprises

The ability to enforce IP allow lists at the user namespace level means that security policies can be more precisely aligned with organizational needs. It exemplifies GitHub's commitment to delivering robust security features tailored for enterprise environments.

## Conclusion

This new feature represents a significant step toward stronger security postures within GitHub Enterprise Cloud, helping organizations safeguard their code and workflows by controlling access through IP-based restrictions at the user namespace level.
